# 农药残留检测APP软件设计说明书

## 1. 系统架构

### 1.1 整体架构
本软件采用分层架构设计，基于Flutter框架开发，主要分为以下几层：

```
┌────────────────────────────────────────────────────────────────────┐
│                           表现层 (UI)                           │
│ ┌─────────────────────┐ ┌─────────────────────┐ ┌──────────────┐ │
│ │     屏幕组件        │ │     通用组件        │ │ 可解释性组件 │ │
│ └─────────────────────┘ └─────────────────────┘ └──────────────┘ │
├────────────────────────────────────────────────────────────────────┤
│                           业务逻辑层                               │
│ ┌─────────────────────┐ ┌─────────────────────┐ ┌──────────────┐ │
│ │    状态管理 (Provider)│ │      服务类        │ │  工具类      │ │
│ └─────────────────────┘ └─────────────────────┘ └──────────────┘ │
├────────────────────────────────────────────────────────────────────┤
│                           数据层                                   │
│ ┌─────────────────────┐ ┌─────────────────────┐ ┌──────────────┐ │
│ │    数据模型         │ │    本地存储        │ │  数据加密    │ │
│ └─────────────────────┘ └─────────────────────┘ └──────────────┘ │
├────────────────────────────────────────────────────────────────────┤
│                         外部接口层                                 │
│ ┌─────────────────────┐ ┌─────────────────────┐ ┌──────────────┐ │
│ │    蓝牙通信         │ │    相机服务        │ │  文件系统    │ │
│ └─────────────────────┘ └─────────────────────┘ └──────────────┘ │
└────────────────────────────────────────────────────────────────────┘
```

### 1.2 技术栈
- **前端框架**：Flutter 3.0+，Dart
- **状态管理**：Provider
- **本地存储**：Hive + SQLite
- **蓝牙通信**：flutter_blue_plus
- **机器学习**：TensorFlow Lite
- **PDF生成**：pdf + printing
- **数据加密**：encrypt + flutter_secure_storage
- **模型训练**：Python，TensorFlow

## 4. 核心功能设计

### 4.1 农药残留检测功能

#### 4.1.1 功能流程
1. **样品信息输入**：用户输入样品名称和类别
2. **检测模式选择**：选择导入模式、模拟模式或实时模式
3. **数据采集**：
   - 导入模式：从文件导入光谱数据
   - 模拟模式：生成模拟光谱数据
   - 实时模式：通过蓝牙设备采集光谱数据
4. **数据预处理**：基线校正、平滑滤波、归一化
5. **AI分析**：
   - 深度学习模式：使用TFLite模型推理
   - 规则引擎模式：基于特征波长匹配
   - 混合模式：融合两种方法的结果
6. **结果处理**：计算风险等级、置信度
7. **结果展示**：显示检测结果和光谱图表
8. **可解释性分析**：分析特征重要性、SHAP值
9. **报告生成**：生成PDF检测报告
10. **结果存储**：保存检测结果和光谱数据

#### 4.1.2 技术实现
- **数据采集**：使用`BluetoothService`管理蓝牙连接和数据传输
- **数据预处理**：使用`EnhancedPreprocessor`进行数据预处理
- **特征工程**：使用`FeatureEngineer`提取特征向量
- **AI分析**：使用`DeepLearningAnalyzer`进行模型推理
- **结果处理**：使用`AIAnalysisService`处理分析结果
- **可解释性分析**：使用`ExplainabilityService`提供AI解释
- **报告生成**：使用`PdfReportService`生成PDF报告
- **结果存储**：使用`StorageService`保存数据

### 4.2 蓝牙设备管理功能

#### 4.2.1 功能流程
1. **蓝牙状态检查**：检查蓝牙是否开启
2. **设备扫描**：扫描附近的蓝牙设备
3. **设备连接**：连接选择的设备
4. **数据传输**：发送命令和接收数据
5. **状态监控**：监控设备连接状态和电量
6. **自动重连**：设备断开时自动尝试重连
7. **设备管理**：保存和管理常用设备

#### 4.2.2 技术实现
- **蓝牙状态管理**：使用`flutter_blue_plus`检查和控制蓝牙状态
- **设备扫描**：使用`FlutterBlue`的扫描功能
- **设备连接**：使用`BluetoothDevice`的连接方法
- **数据传输**：使用`BluetoothCharacteristic`进行数据读写
- **状态监控**：使用Stream监听连接状态变化
- **自动重连**：实现指数退避的重连机制
- **设备管理**：使用`StorageService`保存设备信息

### 4.3 数据存储功能

#### 4.3.1 功能流程
1. **数据验证**：验证数据的完整性和合法性
2. **数据加密**：对敏感数据进行AES加密
3. **数据存储**：将数据存储到Hive和SQLite
4. **数据查询**：支持按条件查询数据
5. **数据备份**：将数据备份为JSON文件
6. **数据恢复**：从备份文件恢复数据
7. **数据导出**：导出数据为CSV格式

#### 4.3.2 技术实现
- **数据验证**：实现数据验证逻辑
- **数据加密**：使用`encrypt`库进行AES加密
- **密钥管理**：使用`flutter_secure_storage`安全存储密钥
- **数据存储**：使用`Hive`存储结构化数据，`SQLite`存储关系数据
- **数据查询**：实现分页查询和条件查询
- **数据备份**：实现JSON序列化和文件操作
- **数据恢复**：实现JSON反序列化和数据导入
- **数据导出**：实现CSV格式转换和文件导出

### 4.4 PDF报告生成功能

#### 4.4.1 功能流程
1. **数据收集**：收集检测结果和相关信息
2. **报告结构**：构建多页报告结构
3. **内容生成**：生成各页面内容
4. **中文字体处理**：加载和应用中文字体
5. **PDF生成**：使用pdf库生成PDF文件
6. **报告保存**：保存PDF到本地存储
7. **报告分享**：支持分享和打印报告

#### 4.4.2 技术实现
- **数据收集**：从`DetectionResult`和`ExplainabilityResult`收集数据
- **报告结构**：设计多页报告模板
- **内容生成**：使用pdf库的widget系统构建内容
- **中文字体处理**：使用Google Fonts的Noto Sans SC
- **PDF生成**：使用`pdf`库的`Document`类
- **报告保存**：使用平台特定的文件IO实现
- **报告分享**：使用`share_plus`库实现分享功能
