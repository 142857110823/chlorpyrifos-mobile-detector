# 依赖管理文档

## 项目依赖概述

本项目使用了多个Flutter和Dart依赖包，这些依赖包为应用提供了各种功能，如状态管理、蓝牙通信、深度学习、数据存储等。

## 核心依赖包

### 1. 状态管理
- **provider** (^6.1.5+1)
  - 用途：状态管理，管理应用的全局状态
  - 版本要求：6.1.0及以上
  - 说明：轻量级状态管理解决方案，适合中小型应用

### 2. 蓝牙通信
- **flutter_blue_plus** (^1.36.8)
  - 用途：蓝牙低功耗(BLE)通信，连接光谱检测设备
  - 版本要求：1.31.0及以上
  - 说明：跨平台蓝牙解决方案，支持Android、iOS、Linux、Windows和macOS

### 3. 网络请求
- **dio** (^5.9.1)
  - 用途：网络请求，用于模型更新、数据同步等
  - 版本要求：5.0.0及以上
  - 说明：功能强大的HTTP客户端，支持拦截器、超时等

### 4. 数据存储
- **hive** (^2.2.3)
  - 用途：轻量级键值存储，存储用户设置、历史记录等
  - 版本要求：2.2.0及以上
  - 说明：高性能NoSQL数据库，适合移动应用

- **hive_flutter** (^1.1.0)
  - 用途：Hive的Flutter集成
  - 版本要求：1.1.0及以上

- **sqflite** (^2.3.3+1)
  - 用途：SQLite数据库，存储结构化数据
  - 版本要求：2.3.0及以上
  - 说明：适合存储复杂的结构化数据，如详细的检测历史

### 5. 图表展示

## 依赖兼容性

### 平台兼容性
- **Android**：所有依赖均支持Android 6.0+ (API 23+)
- **iOS**：所有依赖均支持iOS 12.0+
- **Web**：部分依赖可能在Web平台有限制，特别是蓝牙和TFLite相关功能
- **Windows**：支持Windows 10+
- **Linux**：支持主流Linux发行版

### Dart和Flutter版本要求
- **Dart SDK**：>=3.3.0 <4.0.0
- **Flutter**：>=3.19.0

## 依赖管理最佳实践

### 1. 版本锁定
- 使用`^`符号锁定依赖版本，确保兼容性的同时允许小版本更新
- 定期更新依赖版本，修复安全漏洞和性能问题

### 2. 依赖冲突解决
- 如果遇到依赖冲突，使用`flutter pub outdated`检查过时依赖
- 使用`flutter pub upgrade`更新依赖到最新兼容版本
- 如需要，使用`dependency_overrides`解决冲突

### 3. 依赖分析
- 定期运行`flutter pub deps`分析依赖树
- 移除未使用的依赖，减少应用体积

### 4. 环境配置
- 在CI/CD流程中使用固定的依赖版本
- 使用`pubspec.lock`文件确保团队开发环境一致

## 常见问题及解决方案

### 1. 依赖安装失败
**症状**：`flutter pub get`失败
**解决方案**：
- 检查网络连接
- 删除`pubspec.lock`文件后重新运行`flutter pub get`
- 检查依赖版本是否兼容

### 2. 运行时崩溃
**症状**：应用启动或运行时崩溃
**解决方案**：
- 检查依赖版本是否与Flutter版本兼容
- 查看错误日志，定位具体依赖问题
- 回退到已知工作的依赖版本

### 3. 构建失败
**症状**：`flutter build`失败
**解决方案**：
- 确保所有依赖都支持目标平台
- 检查依赖的原生代码是否需要特殊配置
- 查看构建日志，定位具体错误

## 依赖更新流程

1. **检查过时依赖**：
   ```bash
   flutter pub outdated
   ```

2. **更新依赖**：
   ```bash
   flutter pub upgrade
   ```

3. **验证更新**：
   - 运行应用确保功能正常
   - 运行测试确保没有回归
   - 检查构建是否成功

4. **提交更新**：
   - 提交更新后的`pubspec.yaml`和`pubspec.lock`文件

## 结论

合理的依赖管理对于应用的稳定性、性能和可维护性至关重要。本项目使用了经过验证的依赖版本，确保应用在不同平台和环境下的一致性和可靠性。

通过定期更新依赖、解决冲突、分析依赖树，可以保持应用的健康状态，为用户提供更好的体验。
